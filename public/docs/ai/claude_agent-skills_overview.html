CLAUDE AI

Agent Skills

Claude의 기능을 확장하는 모듈형 기능에 대한 종합 가이드

Agent Skills란? Claude의 기능을 확장하는 모듈형 기능입니다. 각 Skill은 Claude가 관련성이 있을 때 자동으로 사용하는 지침, 메타데이터, 선택적 리소스(스크립트, 템플릿)를 패키징합니다.

Skills를 사용하는 이유

Skills는 범용 에이전트를 전문가로 변환시키는 도메인별 전문 지식(워크플로우, 컨텍스트, 모범 사례)을 Claude에게 제공하는 재사용 가능한 파일시스템 기반 리소스입니다. 프롬프트(일회성 작업을 위한 대화 수준의 지침)와 달리, Skills는 필요할 때 로드되며 여러 대화에서 동일한 가이드를 반복적으로 제공할 필요가 없습니다.

주요 이점

  • Claude 전문화: 도메인별 작업에 맞게 기능 조정
  • 반복 감소: 한 번 만들면 자동으로 사용
  • 기능 조합: Skills를 결합하여 복잡한 워크플로우 구축
Agent Skills의 아키텍처와 실제 응용에 대한 심층 분석은 Anthropic 엔지니어링 블로그를 참조하세요: Equipping agents for the real world with Agent Skills

Skills 사용하기

Anthropic은 일반적인 문서 작업(PowerPoint, Excel, Word, PDF)을 위한 사전 구축된 Agent Skills를 제공하며, 사용자 정의 Skills를 만들 수도 있습니다. 두 가지 모두 동일한 방식으로 작동하며, Claude가 요청과 관련이 있을 때 자동으로 사용합니다.

사전 구축된 Agent Skills

claude.ai와 Claude API의 모든 사용자가 사용할 수 있습니다. 전체 목록은 아래 사용 가능한 Skills 섹션을 참조하세요.

사용자 정의 Skills

도메인 전문 지식과 조직 지식을 패키징할 수 있습니다. Claude의 모든 제품에서 사용 가능합니다: Claude Code에서 생성하거나, API를 통해 업로드하거나, claude.ai 설정에서 추가할 수 있습니다.

시작하기:
  • 사전 구축된 Agent Skills: API에서 PowerPoint, Excel, Word, PDF Skills 사용을 시작하려면 빠른 시작 튜토리얼을 참조하세요
  • 사용자 정의 Skills: 자신만의 Skills를 만드는 방법을 배우려면 Agent Skills Cookbook을 참조하세요

Skills 작동 방식

Skills는 Claude의 VM 환경을 활용하여 프롬프트만으로는 불가능한 기능을 제공합니다. Claude는 파일시스템 액세스가 가능한 가상 머신에서 작동하므로, Skills는 지침, 실행 가능한 코드, 참고 자료가 포함된 디렉토리로 존재할 수 있으며, 새로운 팀원을 위해 만드는 온보딩 가이드처럼 구성됩니다.

이러한 파일시스템 기반 아키텍처는 점진적 공개(progressive disclosure)를 가능하게 합니다: Claude는 미리 컨텍스트를 소비하는 대신 필요에 따라 단계적으로 정보를 로드합니다.

세 가지 유형의 Skill 콘텐츠, 세 가지 로딩 레벨

Skills는 각각 다른 시점에 로드되는 세 가지 유형의 콘텐츠를 포함할 수 있습니다:

레벨 1: 메타데이터 (항상 로드됨)

콘텐츠 유형: 지침. Skill의 YAML 프론트매터는 발견 정보를 제공합니다:

---
name: pdf-processing
description: PDF 파일에서 텍스트와 표 추출, 양식 채우기, 문서 병합. PDF 파일로 작업하거나 사용자가 PDF, 양식, 문서 추출을 언급할 때 사용.
---

Claude는 시작할 때 이 메타데이터를 로드하고 시스템 프롬프트에 포함시킵니다. 이러한 경량 접근 방식은 컨텍스트 페널티 없이 많은 Skills를 설치할 수 있음을 의미합니다. Claude는 각 Skill이 존재한다는 것과 언제 사용해야 하는지만 알고 있습니다.

레벨 2: 지침 (트리거될 때 로드됨)

콘텐츠 유형: 지침. SKILL.md의 본문에는 절차적 지식이 포함됩니다: 워크플로우, 모범 사례, 가이드라인:

# PDF 처리

## 빠른 시작

pdfplumber를 사용하여 PDF에서 텍스트 추출:

```python
import pdfplumber

with pdfplumber.open("document.pdf") as pdf:
    text = pdf.pages[0].extract_text()
```

고급 양식 채우기는 [FORMS.md](FORMS.md)를 참조하세요.

Skill의 설명과 일치하는 요청을 하면, Claude는 bash를 통해 파일시스템에서 SKILL.md를 읽습니다. 그때서야 이 콘텐츠가 컨텍스트 윈도우에 들어갑니다.

레벨 3: 리소스 및 코드 (필요에 따라 로드됨)

콘텐츠 유형: 지침, 코드, 리소스. Skills는 추가 자료를 번들로 제공할 수 있습니다:

pdf-skill/
├── SKILL.md (주요 지침)
├── FORMS.md (양식 채우기 가이드)
├── REFERENCE.md (상세 API 참조)
└── scripts/
    └── fill_form.py (유틸리티 스크립트)

지침: 전문화된 가이드와 워크플로우가 포함된 추가 마크다운 파일 (FORMS.md, REFERENCE.md)

코드: Claude가 bash를 통해 실행하는 실행 가능한 스크립트 (fill_form.py, validate.py); 스크립트는 컨텍스트를 소비하지 않고 결정적 작업을 제공합니다

리소스: 데이터베이스 스키마, API 문서, 템플릿, 예제와 같은 참고 자료

Claude는 참조될 때만 이러한 파일에 액세스합니다. 파일시스템 모델은 각 콘텐츠 유형이 서로 다른 장점을 가진다는 것을 의미합니다: 유연한 가이드를 위한 지침, 신뢰성을 위한 코드, 사실적 조회를 위한 리소스.

레벨 로드 시점 토큰 비용 콘텐츠
레벨 1: 메타데이터 항상 (시작 시) Skill당 ~100 토큰 YAML 프론트매터의 name과 description
레벨 2: 지침 Skill 트리거 시 5k 토큰 미만 지침과 가이드가 포함된 SKILL.md 본문
레벨 3+: 리소스 필요에 따라 사실상 무제한 콘텐츠를 컨텍스트에 로드하지 않고 bash를 통해 실행되는 번들 파일

점진적 공개는 언제든지 관련 콘텐츠만 컨텍스트 윈도우를 차지하도록 보장합니다.

Skills 아키텍처

Skills는 Claude가 파일시스템 액세스, bash 명령, 코드 실행 기능을 갖춘 코드 실행 환경에서 실행됩니다. Skills는 가상 머신의 디렉토리로 존재하며, Claude는 컴퓨터에서 파일을 탐색할 때 사용하는 것과 동일한 bash 명령을 사용하여 상호 작용합니다.

Claude가 Skill 콘텐츠에 액세스하는 방법

Skill이 트리거되면 Claude는 bash를 사용하여 파일시스템에서 SKILL.md를 읽어 컨텍스트 윈도우로 가져옵니다. 해당 지침이 다른 파일(예: FORMS.md 또는 데이터베이스 스키마)을 참조하는 경우, Claude는 추가 bash 명령을 사용하여 해당 파일도 읽습니다. 지침이 실행 가능한 스크립트를 언급하면, Claude는 bash를 통해 실행하고 출력만 받습니다(스크립트 코드 자체는 컨텍스트에 들어가지 않음).

이 아키텍처가 가능하게 하는 것

온디맨드 파일 액세스

Claude는 각 특정 작업에 필요한 파일만 읽습니다. Skill에 수십 개의 참조 파일이 포함되어 있어도, 작업에 sales 스키마만 필요하면 Claude는 해당 파일만 로드합니다. 나머지는 파일시스템에 남아 토큰을 소비하지 않습니다.

효율적인 스크립트 실행

Claude가 validate_form.py를 실행할 때, 스크립트의 코드는 컨텍스트 윈도우에 로드되지 않습니다. 스크립트의 출력("검증 통과" 또는 특정 오류 메시지와 같은)만 토큰을 소비합니다. 이는 Claude가 즉석에서 동등한 코드를 생성하는 것보다 스크립트를 훨씬 더 효율적으로 만듭니다.

번들 콘텐츠에 대한 실질적 제한 없음

파일이 액세스될 때까지 컨텍스트를 소비하지 않기 때문에, Skills는 포괄적인 API 문서, 대규모 데이터세트, 광범위한 예제 또는 필요한 모든 참고 자료를 포함할 수 있습니다. 사용되지 않는 번들 콘텐츠에 대한 컨텍스트 페널티가 없습니다.

이러한 파일시스템 기반 모델이 점진적 공개를 작동하게 만드는 것입니다. Claude는 온보딩 가이드의 특정 섹션을 참조하듯이 Skill을 탐색하며, 각 작업이 필요로 하는 것만 정확히 액세스합니다.

예시: PDF 처리 Skill 로딩

다음은 Claude가 PDF 처리 Skill을 로드하고 사용하는 방법입니다:

  1. 시작: 시스템 프롬프트에 포함: "PDF Processing - PDF 파일에서 텍스트와 표 추출, 양식 채우기, 문서 병합"
  2. 사용자 요청: "이 PDF에서 텍스트를 추출하고 요약해줘"
  3. Claude 호출: bash: read pdf-skill/SKILL.md → 지침이 컨텍스트에 로드됨
  4. Claude 판단: 양식 채우기가 필요하지 않으므로 FORMS.md는 읽지 않음
  5. Claude 실행: SKILL.md의 지침을 사용하여 작업 완료

다이어그램은 다음을 보여줍니다:

  1. 시스템 프롬프트와 Skill 메타데이터가 미리 로드된 기본 상태
  2. Claude가 bash를 통해 SKILL.md를 읽어 Skill을 트리거
  3. Claude가 필요에 따라 FORMS.md와 같은 추가 번들 파일을 선택적으로 읽음
  4. Claude가 작업을 진행

이러한 동적 로딩은 관련 Skill 콘텐츠만 컨텍스트 윈도우를 차지하도록 보장합니다.

Skills가 작동하는 곳

Skills는 Claude의 에이전트 제품 전반에서 사용할 수 있습니다:

Claude API

Claude API는 사전 구축된 Agent Skills와 사용자 정의 Skills를 모두 지원합니다. 둘 다 동일하게 작동합니다: 코드 실행 도구와 함께 container 매개변수에서 관련 skill_id를 지정하세요.

전제 조건: API를 통해 Skills를 사용하려면 세 가지 베타 헤더가 필요합니다:
  • code-execution-2025-08-25 - Skills는 코드 실행 컨테이너에서 실행됩니다
  • skills-2025-10-02 - Skills 기능을 활성화합니다
  • files-api-2025-04-14 - 컨테이너로/에서 파일 업로드/다운로드에 필요합니다

사전 구축된 Agent Skills는 skill_id(예: pptx, xlsx)를 참조하여 사용하거나, Skills API(/v1/skills 엔드포인트)를 통해 직접 만들고 업로드할 수 있습니다. 사용자 정의 Skills는 조직 전체에서 공유됩니다.

Claude Code

Claude Code는 사용자 정의 Skills만 지원합니다.

사용자 정의 Skills: SKILL.md 파일이 있는 디렉토리로 Skills를 만듭니다. Claude가 자동으로 발견하고 사용합니다.

Claude Code의 사용자 정의 Skills는 파일시스템 기반이며 API 업로드가 필요하지 않습니다.

Claude Agent SDK

Claude Agent SDK는 파일시스템 기반 구성을 통해 사용자 정의 Skills를 지원합니다.

사용자 정의 Skills: .claude/skills/에 SKILL.md 파일이 있는 디렉토리로 Skills를 만듭니다. allowed_tools 구성에 "Skill"을 포함하여 Skills를 활성화합니다.

SDK의 Skills는 SDK가 실행될 때 자동으로 발견됩니다.

Claude.ai

Claude.ai는 사전 구축된 Agent Skills와 사용자 정의 Skills를 모두 지원합니다.

사전 구축된 Agent Skills: 이러한 Skills는 문서를 만들 때 이미 백그라운드에서 작동하고 있습니다. Claude는 설정 없이 자동으로 사용합니다.

사용자 정의 Skills: 설정 > 기능을 통해 자신의 Skills를 zip 파일로 업로드합니다. 코드 실행이 활성화된 Pro, Max, Team, Enterprise 플랜에서 사용할 수 있습니다. 사용자 정의 Skills는 각 사용자별로 개별적이며, 조직 전체에서 공유되지 않고 관리자가 중앙에서 관리할 수 없습니다.

Skill 구조

모든 Skill에는 YAML 프론트매터가 있는 SKILL.md 파일이 필요합니다:

---
name: your-skill-name
description: 이 Skill이 하는 일과 언제 사용해야 하는지에 대한 간단한 설명
---

# Your Skill Name

## 지침
[Claude가 따를 명확하고 단계별 가이드]

## 예제
[이 Skill 사용에 대한 구체적인 예제]

필수 필드

name과 description

필드 요구사항

name:

  • 최대 64자
  • 소문자, 숫자, 하이픈만 포함해야 함
  • XML 태그를 포함할 수 없음
  • 예약어를 포함할 수 없음: "anthropic", "claude"

description:

  • 비어 있지 않아야 함
  • 최대 1024자
  • XML 태그를 포함할 수 없음

description에는 Skill이 하는 일과 Claude가 언제 사용해야 하는지가 모두 포함되어야 합니다. 완전한 작성 가이드는 모범 사례 가이드를 참조하세요.

보안 고려사항

신뢰할 수 있는 출처(직접 만들었거나 Anthropic에서 입수한)의 Skills만 사용하는 것을 강력히 권장합니다. Skills는 지침과 코드를 통해 Claude에게 새로운 기능을 제공하며, 이것이 강력한 이유이지만, 악의적인 Skill이 Skill의 명시된 목적과 일치하지 않는 방식으로 도구를 호출하거나 코드를 실행하도록 Claude를 지시할 수 있음을 의미합니다.

경고: 신뢰할 수 없거나 알 수 없는 출처의 Skill을 사용해야 하는 경우, 극도로 주의하고 사용하기 전에 철저히 감사하세요. Claude가 Skill을 실행할 때 어떤 액세스 권한을 갖는지에 따라, 악의적인 Skills는 데이터 유출, 무단 시스템 액세스 또는 기타 보안 위험으로 이어질 수 있습니다.

주요 보안 고려사항

  • 철저히 감사: Skill에 번들로 제공되는 모든 파일(SKILL.md, 스크립트, 이미지, 기타 리소스)을 검토합니다. 예상치 못한 네트워크 호출, 파일 액세스 패턴 또는 Skill의 명시된 목적과 일치하지 않는 작업과 같은 비정상적인 패턴을 찾으세요
  • 외부 소스는 위험: 외부 URL에서 데이터를 가져오는 Skills는 특히 위험합니다. 가져온 콘텐츠에 악의적인 지침이 포함될 수 있기 때문입니다. 신뢰할 수 있는 Skills도 외부 종속성이 시간이 지남에 따라 변경되면 손상될 수 있습니다
  • 도구 오용: 악의적인 Skills는 해로운 방식으로 도구(파일 작업, bash 명령, 코드 실행)를 호출할 수 있습니다
  • 데이터 노출: 민감한 데이터에 액세스할 수 있는 Skills는 외부 시스템으로 정보를 유출하도록 설계될 수 있습니다
  • 소프트웨어 설치처럼 취급: 신뢰할 수 있는 출처의 Skills만 사용하세요. 민감한 데이터나 중요한 작업에 액세스할 수 있는 프로덕션 시스템에 Skills를 통합할 때 특히 주의하세요

사용 가능한 Skills

사전 구축된 Agent Skills

다음 사전 구축된 Agent Skills를 즉시 사용할 수 있습니다:

  • PowerPoint (pptx): 프레젠테이션 생성, 슬라이드 편집, 프레젠테이션 콘텐츠 분석
  • Excel (xlsx): 스프레드시트 생성, 데이터 분석, 차트가 있는 보고서 생성
  • Word (docx): 문서 생성, 콘텐츠 편집, 텍스트 서식 지정
  • PDF (pdf): 서식이 지정된 PDF 문서 및 보고서 생성

이러한 Skills는 Claude API와 claude.ai에서 사용할 수 있습니다. API에서 사용하기 시작하려면 빠른 시작 튜토리얼을 참조하세요.

사용자 정의 Skills 예제

사용자 정의 Skills의 완전한 예제는 Skills cookbook을 참조하세요.

제한 사항 및 제약

이러한 제한 사항을 이해하면 Skills 배포를 효과적으로 계획하는 데 도움이 됩니다.

플랫폼 간 가용성

사용자 정의 Skills는 플랫폼 간에 동기화되지 않습니다. 한 플랫폼에 업로드된 Skills는 다른 플랫폼에서 자동으로 사용할 수 없습니다:

  • Claude.ai에 업로드된 Skills는 API에 별도로 업로드해야 합니다
  • API를 통해 업로드된 Skills는 Claude.ai에서 사용할 수 없습니다
  • Claude Code Skills는 파일시스템 기반이며 Claude.ai 및 API와 별개입니다

공유 범위

Skills는 사용하는 위치에 따라 다른 공유 모델을 갖습니다:

  • Claude.ai: 개별 사용자만; 각 팀원이 별도로 업로드해야 합니다
  • Claude API: 작업 공간 전체; 모든 작업 공간 구성원이 업로드된 Skills에 액세스할 수 있습니다
  • Claude Code: 개인(~/.claude/skills/) 또는 프로젝트 기반(.claude/skills/); Claude Code 플러그인을 통해 공유할 수도 있습니다

Claude.ai는 현재 사용자 정의 Skills의 중앙 집중식 관리자 관리 또는 조직 전체 배포를 지원하지 않습니다.

런타임 환경 제약

Skill에서 사용할 수 있는 정확한 런타임 환경은 사용하는 제품 플랫폼에 따라 다릅니다.

Claude.ai:

  • 다양한 네트워크 액세스: 사용자/관리자 설정에 따라 Skills는 전체, 부분 또는 네트워크 액세스가 없을 수 있습니다

Claude API:

  • 네트워크 액세스 없음: Skills는 외부 API 호출을 하거나 인터넷에 액세스할 수 없습니다
  • 런타임 패키지 설치 없음: 사전 설치된 패키지만 사용할 수 있습니다. 실행 중에 새 패키지를 설치할 수 없습니다
  • 사전 구성된 종속성만: 사용 가능한 패키지 목록은 코드 실행 도구 문서를 확인하세요

Claude Code:

  • 전체 네트워크 액세스: Skills는 사용자 컴퓨터의 다른 프로그램과 동일한 네트워크 액세스 권한을 갖습니다
  • 전역 패키지 설치 권장하지 않음: Skills는 사용자 컴퓨터를 방해하지 않도록 로컬에만 패키지를 설치해야 합니다

이러한 제약 조건 내에서 작동하도록 Skills를 계획하세요.
